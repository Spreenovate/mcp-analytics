# See https://www.robotstxt.org/robotstxt.html
#
# Policy: index all public content (landing, docs, blog, /vs, /mcp/tools).
# Keep authenticated/transactional surfaces (/signup, /verify, /settings,
# /oauth/*) and the JSON-RPC endpoint (/mcp itself) out of the index.
#
# AI crawlers are explicitly ALLOWED on the public content surface — we
# want to be discoverable in Claude, ChatGPT, Perplexity, and other
# LLM-grounded answer surfaces, that's a primary acquisition channel
# for an MCP-native product. Each named-bot block re-declares the same
# Disallow set because per robots.txt RFC the named-UA block REPLACES
# the wildcard block, it does not extend it; missing Disallow lines
# under a named bot would otherwise let GPTBot etc. crawl /oauth/* and
# /verify/*.
#
# The "Allow: /mcp/tools" line ahead of "Disallow: /mcp$" is for
# crawlers that don't honor the $-suffix wildcard (Yandex, some smaller
# bots) — they read /mcp as a path prefix and would otherwise block
# /mcp/tools too. The $-honoring crawlers (Google, Bing) treat the
# Allow line as a no-op since /mcp/tools is already not matched by
# /mcp$.
